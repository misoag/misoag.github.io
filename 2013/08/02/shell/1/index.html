管道和重定向

  • 0 代表标准输入
  • 1 代表标准输出
  • 2 代表标准错误输出

重定向输出

#将标准输入和标准输出分别重定向到不同的文件中
$ kill -HUP 1234 >killout.txt 2>killerr.txt

#将两组输出都重定向到一个文件中，可以使用&>操作符来结合两个输出
$ kill -1 1234 >killout.txt 2&>1

#使用Linux回收站丢弃所有的输出信息
$ kill -1 1234 >/dev/null 2&>1

重定向输入

$ more < killout.txt

shell 环境变量

$HOME       当前用户的家目录
$PATH 
$PS1        命令提示符，通常是$。 也可以设置更复杂的值，如：[\u@\h \W]$，给出了用户名、机器名、当前目录、
$PS2        二级目录用来提示后续的输入，通常是>字符
$IFS        输入域分隔符。当shell读取输入时，它给出用来分割单词的一组字符，他们通常是空格、制表符和换行符。
$0          shell脚本的名字
$#          传递给脚本的参数个数
$$          shell脚本的进程号，脚本程序通常会用它来生成一个唯一的临时文件，如 /tmp/tmpfile_$$

参数变量

$1,$2,...   脚本程序的参数
$*          在一个变量中列出所有的参数，各参数之间用环境变量 IFS 中的第一个字符分隔开。
$@          它是$*的一个变种，即使IFS位空，参数也不会挤在一起。

通过下面的例子，你可以很容易的看出$@和$*之间的区别：

macbook:~ john$ IFS=''
macbook:~ john$ set foo bar bam
macbook:~ john$ echo "$@"
foo bar bam
macbook:~ john$ echo "$*"
foobarbam
macbook:~ john$ unset IFS
macbook:~ john$ echo "$*"
foo bar bam

一般来说，如果你想访问程序的参数，使用$@是明智的选择。

条件测试

test 或者 [ 命令

if test -f fred.c
then
...
fi

也可以写成下面这样：

if [ -f fred.c ]
then
...
fi

也可以写成这样：

if [ -f fred.c ];then
...
fi

字符串比较

string1 = string2       如果两个字符串相同则为真
string1 != string2      如果两个字符串不同则为真
-n string               如果字符串不为空则为真
-z string               如果字符串为null（一个空串）则为真

算数比较

exp1 -eq exp2   两个表达式相等则为真
exp1 -ne exp2   两个表达式不等则为真
exp1 -gt exp2   exp1 大于 exp2 则为真
exp1 -ge exp2   exp1 大于等于 exp2 则为真
exp1 -lt exp2   exp1 小于 exp2 则为真
exp1 -le exp2   exp1 小于等于 exp2 则为真
! exp           表达式为假则为真（反之亦然）

文件测试

-d file     如果文件是一个目录则为真
-e file     如果文件存在则为真。 注意：历史上-e选项不可移植，所以，我们通常使用-f选项
-f file     如果文件存在则为真
-g file     如果文件的 set-group-id 位被设置则为真
-u file     如果文件的 set-user-id 位被设置则为真
-s file     如果文件大小不为0则为真
-r file     如果文件可读则为真
-w file     如果文件可写则为真
-x file     如果文件可执行则为真

#!/bin/sh

if [ -f /bin/bash ]
then
    echo "file /bin/bash exists"
fi

if [ -d /bin/bash ]
then
    echo "/bin/bash is a directory"
else
    echo "/bin/bash is NOT a directory"
fi

控制结构

if..elif..else…fi语句

#!/bin/sh
echo "Is it morning? Please answer yes or no"
read timeofday

if [ "$timeofday" == "yes" ]
then
    echo "Good morning"
elif [ "$timeofday" = "no" ];then
    echo "Good afternoon"
else
    echo "Sorry, $timeofday not recognized. Enter yes or no"
    exit 1
fi

exit 0

for 语句

实验1：

#!/bin/sh
for foo in bar fud 43
do
    echo $foo
done
exit 0

实验2：

#!/bin/sh
for line in $(ls -a);do
    echo $line
done
exit 0

while语句

#!/bin/sh
echo "Enter password"
read trythis
while [ "$trythis" != "secret" ];do
    echo "Sorry,try again"
    read trythis
done
exit 0

until语句

until condition
do
    statements
done

case语句

case variable in
    pattern [ | pattern] ...) statements;;
    pattern [ | pattern] ...) statements;;
esac

实验1

#!/bin/sh
echo "Is it morning? Please answer yes or no"
read timeofday
case "$timeofday" in
    yes )   echo "Good Morning";;
    no  )   echo "Good Afternoon";;
    y   )   echo "Good Morning";;
    n   )   echo "Good Afternoon";;
    *   )   echo "Sorry, answer not recognized";;
esac
exit 0

实验2

#!/bin/sh
echo "Is it morning? Please answer yes or no"
read timeofday
case "$timeofday" in
    y | yes | Y | Yes | YES )   echo "Good Morning";;
    n* | N*                 )   echo "Good Afternoon";;
    *                       )   echo "Sorry, answer not recognized";;
esac
exit 0

AND列表

#!/bin/sh
touch file_one
rm -f file_two
if [ -f file_one ] && echo "hello" && [ -f file_two ] && echo "there"
then
    echo "in if"
else
    echo "in else"
fi
exit 0

OR列表

#!/bin/sh
rm -f file_one
if [ -f file_one ] || echo "hello" || echo "there"
then
    echo "in if"
else
    echo "in else"
fi
exit 0

解析：第一条指令if [ -f file_one ]失败了，但是接下来的echo语句，它返回的是true。

函数

#!/bin/sh
foo() {
    echo "Function foo is executing"
}
echo "script starting"
foo
echo "script ended"
exit 0
