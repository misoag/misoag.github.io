一、程序的真正入口

main或WinMain只是“语法规定的程序入口” 并不是“应用程序入口”。

我们使用VC++ 6.0 的栈回溯功能，找到main函数之前的代码。菜单View -> Debug Windows -> Call Stack 打开栈回溯窗口（快捷键 Alt + 7）。

上图显示程序运行时调用了三个函数，分别为：KERNEL32.DLL、mainCRTStartup和main。 其中KERNEL32! 7C81776F 表示在系统库KERNEL32.dll中的地址7c81776f处调用了mainCRTStartup。
通过双击 mainCRTStartup，我们可以找到这个函数定义在 crt0.c 中，你可以在crt0.c中找到mainCRTStartup的源码。

下面是crt0.c中注释的摘抄：

/***
*crt0.c - C runtime initialization routine
*
*       Copyright (c) Microsoft Corporation. All rights reserved.
*
*Purpose:
*       This is the actual startup routine for apps.  It calls the user's main
*       routine [w]main() or [w]WinMain after performing C Run-Time Library
*       initialization.
*
*       With ifdefs, this source file also provides the source code for:
*       wcrt0.c     the startup routine for console apps with wide chars
*       wincrt0.c   the startup routine for Windows apps
*       wwincrt0.c  the startup routine for Windows apps with wide chars
*
*******************************************************************************/

这里说明了函数真正的入口

/***
*mainCRTStartup(void)
*wmainCRTStartup(void)
*WinMainCRTStartup(void)
*wWinMainCRTStartup(void)
*
*Purpose:
*       These routines do the C runtime initialization, call the appropriate
*       user entry function, and handle termination cleanup.  For a managed
*       app, they then return the exit code back to the calling routine, which
*       is the managed startup code.  For an unmanaged app, they call exit and
*       never return.
*
*       Function:               User entry called:
*       mainCRTStartup          main
*       wmainCRTStartup         wmain
*       WinMainCRTStartup       WinMain
*       wWinMainCRTStartup      wWinMain
*
*Entry:
*
*Exit:
*       Managed app: return value from main() et al, or the exception code if
*                 execution was terminated by the __except guarding the call
*                 to main().
*       Unmanaged app: never return.
*
*******************************************************************************/

由于我安装的的VC++6.0不是完整版，未找到crt0.c。然后，我切换到vs2010下，找到了crt0.c，其中有这样一段定义：

#else  /* _WINMAIN_ */

    #ifdef WPRFLAG

经常看到有人讨论，是switch效率高还是if…else if效率高，今天几个来探寻个究竟。

1、switch 与 if…else if 反汇编比较

一个标准的switch结构：

int _tmain(int argc, _TCHAR* argv[])
{
    switch(argc)
    {
    case 1:
        printf("The Number 1");
        break;
    case 2:
        printf("The Number 2");
        break;
    default:
        printf("default");
        break;
    }
    return 0;
}

通过上图我们可以看出，一共进行了2次比较，5次跳转。 下面我们来实现一段同样功能的if…else if结构：

int _tmain(int argc, _TCHAR* argv[])
{
    if(argc == 1){
        printf("The Number 1");
    }else if(argc == 2){
        printf("The Number 2");
    }else{
        printf("default");
    }
    return 0;
}

我们可以看到，if…else if 结构会在条件跳转后紧跟语句块；而Switch结构则将所有的条件跳转都放置在了一起，并未发现case语句块的踪迹。通过条件跳转指令，跳转到相应的case块中。

2、可构成线性跳转表的switch

【case地址表】

通过查表（数组访问），我们看到数组的第一个元素就是case1的首地址。以此类推，通过数组下标访问，我们可以更快速的找到其他case地址。

如果能构成线性表，此时switch将远远超过if…else if的效率。

总结

mov        reg, mem                  ;取变量
;对变量进行运算，对齐case地址表的0下标，非必要
lea        eax, [reg + xxx]          ;eax 为数组下标
;影响标志位的指令，进行范围检查
jxx        DEFAULT_ADDR
jmp        dword ptr [eax*4 + xxxx]  ;地址xxxx为线性跳转表的首地址

根据跳转表中存放的地址，可以找到每个case的首地址，即可反推处线性关系的switch分支结构。

3、不能构成跳转表的switch（索引表）

在现实中，并非所有的switch都是有序线性的，当两个case的值间隔较大时，这样就会造成空间的巨大浪费。

对于非线性的switch结构，可以采用制作索引表的方法来进行优化。索引表优化需要两张表：case语句块地址表、case语句块索引表。

地址表中的每一项，保存case语句块的首地址，有几个case语句就有几项。default也在其中，如果没有，则保存一个switch的结束地址。

索引表中保存地址表的编号，它的大小等于 最大case值 减去 最小case值 的差值。 当差值大于255时，这种优化方案也会浪费空间。可以使用树方式优化。

索引表大小 = (MAX - MIN) * 1字节    //MAX=case的最大值，MIN=case的最小值
地址表大小 = SUM * 4字节            //SUM为case的数量
总占用字节 = 索引表大小 + 地址表大小

看下面这张图，演示了索引表与地址表的对应关系：

仔细看，不要害怕，没那么复杂，下面我们用代码来演示：

类中不能定义自身的对象，因为类在实例化过程中，需要申请内存，也就是说需要计算自身的大小。如果在类中定义了自身的对象，在计算各成员长度时，就会形成递归，而且是没有出口的递归，将会无限循环。

逆向基础第十七篇：数组和指针

By John·Huang

发表于 2015-05-23

数组和指针都是针对地址进行操作，但是他们有很多不同之处。数组是相同类型的数据集合，是以线性的方式存储在内存中；而指针是保存4个字节的地址变量。

数组名是一个地址常量，保存数组第一个元素的地址（不可以更改），只能作为基址访问内存数据；
指针是一个变量，只要修改所保存的地址数据，就可以随意访问，不受约束。

1、数组的定义

int _tmain(int argc, _TCHAR* argv[])
{
    int aArray[5] = {1,2,3,4,5};
    int var1 = 1;
    int var2 = 2;
    int var3 = 3;
    int var4 = 4;
    int var5 = 5;
}

反汇编结果：

2、数组作为参数

数组所占的内存，通常大于4个字节，那么它是如何传递参数的呢？
测试代码如下：

void Show(char szBuff[])
{
    strcpy(szBuff,"hello world");
    printf(szBuff);
}

int _tmain(int argc, _TCHAR* argv[])
{
    char szHello[20] = {0};
    Show(szHello);
}

反汇编代码：

通过上图，可以看出，当数组作为参数时，传递的是数组的首地址（指针变量）。因为是参数是指针变量，因此，使用sizeof只能得到指针长度，因此，应该避免如下错误代码：

void Show(char szBuff[])
{
    int nLen = 0;

    //这里是错误代码，得到的是指针长度,4字节
    nLen =  sizeof(szBuff);
    printf("%d ,",nLen);

    //正确的使用方法
    nLen = strlen(szBuff);
    printf("%d ,",nLen);

    getchar();
}

3、数组作为返回值

数组作为返回值时，与作为参数时相同，都是返回数组第一个元素的地址。注意，数组作为参数时，其定义的作用域必然在函数外，但是作为返回值时，如果数组在函数内定义，作为函数的局部变量，那么开辟的栈空间，在函数结束的时候要进行堆栈平衡，esp归位。这样，函数所在的内存空间就会极不稳定，随时可能会被破坏，影响程序的结果。
因此，如果想使用数组作为返回值，可以使用“全局数组，静态数组，或者上层调用函数中定义的局部数组”

逆向基础第十六篇：堆空间的申请与释放

逆向基础第十五篇：函数调用的三种约定方式（堆栈平衡）

By John·Huang

发表于 2015-05-23

VC++ 环境下的的函数调用约定有三种：

  • _cedcl C/C++的默认调用方式，调用方平衡栈，不定长参数的函数可以使用。
  • _stdcall 被调用方平衡栈，不定长参数的函数无法使用
  • _fastcall 寄存器方式传参，被调用方平衡栈，不定长参数的函数无法使用。

1、_cdecl 与 _stdcall的对比

void _stdcall ShowStd(int nNumber)
{
    printf("%d",nNumber);
}

void _cdecl ShowCde(int nNumber)
{
    printf("%d",nNumber);
}

int _tmain(int argc, _TCHAR* argv[])
{
    ShowStd(5);//被调用方平衡栈
    ShowCde(5);//调用方平衡栈
}

逆向基础第十四篇：认识栈帧（堆栈平衡）

By John·Huang

发表于 2015-05-23

当栈顶指针esp小于栈底指针ebp时，就形成了栈帧。在VC++中，栈帧用来存放局部变量、函数返回值地址、函数参数等。

当A函数调用B函数时，就会针对B函数开辟出所需的栈空间，形成此B函数的栈帧。当这个B函数结束调用时，要清除开辟出的栈空间，这一过程，称作 “堆栈平衡”。

下面是一个空函数：

int _tmain(int argc, _TCHAR* argv[])
{
    return 0;
}

反汇编代码如下：

int _tmain(int argc, _TCHAR* argv[])
{
00F01350  push        ebp                  ;保存寄存器ebp的值
00F01351  mov         ebp,esp              ;调整当前栈底指针到栈顶（将ebp作为栈底指针）
00F01353  sub         esp,0C0h             ;分配192字节的空间
00F01359  push        ebx                  ;保存寄存器ebx的值
00F0135A  push        esi                  ;保存寄存器esi的值
00F0135B  push        edi                  ;保存寄存器edi的值
00F0135C  lea         edi,[ebp-0C0h]       ;取栈顶地址，保存在edi中
00F01362  mov         ecx,30h  
00F01367  mov         eax,0CCCCCCCCh  
00F0136C  rep stos    dword ptr es:[edi]  
    return 0;
00F0136E  xor         eax,eax              ;对返回值eax清零
}
00F01370  pop         edi                  ;还原寄存器edi的值
00F01371  pop         esi                  ;还原寄存器esi的值
00F01372  pop         ebx                  ;还原寄存器ebx的值
00F01373  mov         esp,ebp              ;还原寄存器esp的值
00F01375  pop         ebp                  ;还原寄存器ebp的值
00F01376  ret
  • rep指令的目的是重复其上面的指令。
  • ecx的值是重复的次数。
  • stos指令的作用是，将eax中的值拷贝到es:[edi]指向的地址。

逆向基础第十三篇：编译器对循环的优化

By John·Huang

发表于 2015-05-23

之前，我已经对do、while、和for循环做了详细的介绍。

do循环：http://www.0x520.com/2014/07/04/247.html
while循环：http://www.0x520.com/2014/07/04/251.html
for循环：http://www.0x520.com/2014/07/05/264.html

通过对比，我们已经知道，do循环的速度最快，while次之，for的速度是最慢的。 因为，do只用到一个跳转指令，while用到两个跳转指令，而for用到了三个跳转指令。

1、优化while循环

在 while循环 中我们讲过，是可以将while转换为do循环的，可以将do循环结构嵌套在if中，来完成while结构的优化。
while循环代码：

void loop_while(int loopTimes)
{
    while(loopTimes>=0)
    {
        //do something
        loopTimes --;
    }
}

优化代码：

void loop_do(int loopTimes)
{
    if(loopTimes>=0)
    {
        do{
            //do something
            loopTimes --;
        }while(loopTimes>=0);
    }
}

如果编译器使用O2选项进行优化，那么，反汇编之后的代码将是上面这种形式。

2、优化for循环

for循环，从结构上看，其结构特征和while循环类似。由于赋初值部分不属于循环体，可以忽略。只要将比较部分放入循环体，那么就是一个while循环。既然可以转换成while循环，那么必然也可以转换成do循环。

for循环代码：

int nSum = 0;
for (int i = 0;i < loop_times ; i++)
{
    nSum += i;
}

for转换成while循环：

int i = 0;
while(i<loop_times){
    nSum += i;
    i++;
}

while循环转换成do循环：

int nSum = 0;
int i = 0;
if(i<loop_times){
    do {
        nSum += i;
        i++;
    } while (i<argc);
}

编译器真的会这样优化吗？我们打开O2选项（选择Release，默认就是O2）：

编译如下代码：

int loop_for(int loop_times)
{
    int nSum = 0;
    for (int i = 0;i < loop_times ; i++)
    {
        nSum += i;
    }
    return nSum;
}

我们使用IDA分析：

上图中for循环优化之后的结构一目了然，但是有些朋友表示还是看不懂，表示对下面这段代码不知所云：

add     edx,eax
lea     ecx,[ecx+eax+1]
add     eax,2
cmp     eax,esi
jle     short loc_401020

这里我来讲解一下，其实这已经不是编译器对for循环优化的范畴了，这里是编译器对当前算法的优化 nSum += (i++);

  • edx 等于变量 nSum 的偶数和
  • ecx 等于变量 nSum 的奇数和
  • eax 等于变量 i

最后，在地址00401034处，lea eax,[ecx+edx]，这里 eax = 奇数和+偶数和 。

for 循环是三种循环（do，while，for）中最复杂的一种。for循环由赋初值、设置循环条件、设置循环步长 这三条语句组成。在程序中for被使用的频率是最高的。

Step3：循环判定部分，然后执行代码块，向上跳跳到Step2

逆向基础第十一篇：控制流程 之 while循环

By John·Huang

发表于 2015-05-23

while循环与do循环正好相反，先执行判定条件，再执行代码块。

测试代码：

int _tmain(int argc, _TCHAR* argv[])
{
    int nSum    = 0;
    int nIndex    = 0;
    while (nIndex <= argc)
    {
        nSum += nIndex;
        nIndex ++;
    } 
    return 0;
}

反汇编代码：

我们看到while循环使用了两次跳转指令完成循环，由于多使用了一次跳转指令，因此while循环要比do循环效率低一些。

总结：

WHILE_BEGIN
;影响标记位的指令
jxx    WHILE_END
......
jmp    WHILE_BEGIN
WHILE_END

遇到以上代码块，即可判断为while循环结构。根据跳转指令的位置，可以还原while循环判断。

注意：while循环结构很可能被优化成do循环结构，被转换后的while结构由于需要检查是否可以被成功执行一次，通常会被潜逃在if单分支结构中，高级代码如下：

if(xxx)
{
    do
    {
        //循环代码块
    } while (xxx)
}
