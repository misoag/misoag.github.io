逆向基础第十二篇：控制流程 之 for循环

By John·Huang

发表于 2014-05-12

for 循环是三种循环（do，while，for）中最复杂的一种。for循环由赋初值、设置循环条件、设置循环步长 这三条语句组成。在程序中for被使用的频率是最高的。

for循环测试代码：

int _tmain(int argc, _TCHAR* argv[])
{
    int nSum = 0;
    for (int i = 0;i < argc ; i++)
    {
        nSum += i;
    }
    printf("%d \r\n",nSum);
    return 0;
}

反汇编代码如下：

通过上图，我们看出，赋初值只执行一次。然后不断自增、执行代码块，往上跳。通过循环流程，我们可以总结其特性。

总结：

Step1：赋初值，跳转到循环判定部分，然后执行代码块

Step2：代码块末尾，向上跳，跳转到自增（修改步长部分，这里是自增，视实际情况而定）
Step3：循环判定部分，然后执行代码块，向上跳跳到Step2

逆向基础第十一篇：控制流程 之 while循环

By John·Huang

发表于 2014-05-11

while循环与do循环正好相反，先执行判定条件，再执行代码块。

测试代码：

int _tmain(int argc, _TCHAR* argv[])
{
    int nSum    = 0;
    int nIndex    = 0;
    while (nIndex <= argc)
    {
        nSum += nIndex;
        nIndex ++;
    } 
    return 0;
}

反汇编代码：

我们看到while循环使用了两次跳转指令完成循环，由于多使用了一次跳转指令，因此while循环要比do循环效率低一些。

总结：

WHILE_BEGIN
;影响标记位的指令
jxx    WHILE_END
......
jmp    WHILE_BEGIN
WHILE_END

遇到以上代码块，即可判断为while循环结构。根据跳转指令的位置，可以还原while循环判断。

注意：while循环结构很可能被优化成do循环结构，被转换后的while结构由于需要检查是否可以被成功执行一次，通常会被潜逃在if单分支结构中，高级代码如下：

if(xxx)
{
    do
    {
        //循环代码块
    } while (xxx)
}

逆向基础第十篇：控制流程 之 do循环

By John·Huang

发表于 2014-05-10

do循环工作流程清晰，识别相对简单。根据do循环的特性：先执行语句块，再进行比较判断。条件成立时，会继续执行语句块。C++中的goto也可以模拟do循环。

测试代码：

int _tmain(int argc, _TCHAR* argv[])
{
    int nSum    = 0;
    int nIndex    = 0;
    do 
    {
        nSum += nIndex;
        nIndex ++;
    } while (nIndex <= argc);
}

反汇编结果：

这与if分支结构非常相似，但是自己分析之后发现他们并不相同，这跟if语句的比较是相反的，if的跳转地址大于当前地址，是一个向下跳的过程。而do的跳转地址小于当前代码地址，是向上跳的。 根据这个特性，我们可以很容易的区分 if 和 do 的反汇编代码。

总结：

DO_BEGIN
......            ;循环语句块
;影响标记位的指令
jxx     DO_BEGIN

如果遇到上面这种代码块，可以判定为是一个do循环。根据跳转到的地址，可以得知代码块的首地址，jxx指令的地址为循环的结束地址。根据这些特征，即可还原do循环结构的原型。

逆向基础第九篇：流程控制 之 switch 判定树

By John·Huang

发表于 2014-05-09

上一篇我们探究了switch的非线性索引表的优化，讲述了 case最大值 和 case最小值的差值在255以内的情况。

但是，当差值大于255时，这种优化方案会造成空间浪费。此时编译器采用另外一种优化方案——判定树。

判定树： 每个case作为一个节点，从这些节点中找出一个中间值作为根节点，以此形成一个平衡二叉树，以每个节点为判定值，大于和小于关系分别对应左子树和右子树，这样可以提高检索效率。

我们构造了这段代码：

switch(argc)
{
case 2: printf("The Number 1"); break;
case 3: printf("The Number 2"); break;
case 8: printf("The Number 3"); break;
case 10: printf("The Number 5"); break;
case 35: printf("The Number 6"); break;
case 37: printf("The Number 6"); break;
case 666: printf("The Number 255"); break;
}

1、switch 与 if…else if 反汇编比较

一个标准的switch结构：

        printf("The Number 2");
        break;
    default:
        printf("default");
        break;
    }

3、不能构成跳转表的switch（索引表）

在现实中，并非所有的switch都是有序线性的，当两个case的值间隔较大时，这样就会造成空间的巨大浪费。

对于非线性的switch结构，可以采用制作索引表的方法来进行优化。索引表优化需要两张表：case语句块地址表、case语句块索引表。

地址表中的每一项，保存case语句块的首地址，有几个case语句就有几项。default也在其中，如果没有，则保存一个switch的结束地址。

索引表中保存地址表的编号，它的大小等于 最大case值 减去 最小case值 的差值。 当差值大于255时，这种优化方案也会浪费空间。可以使用树方式优化。

索引表大小 = (MAX - MIN) * 1字节    //MAX=case的最大值，MIN=case的最小值
地址表大小 = SUM * 4字节            //SUM为case的数量
总占用字节 = 索引表大小 + 地址表大小

By John·Huang

发表于 2014-05-07

这篇文章中的代码是通过vs2010编译的，可能与你反汇编出来的结果不一致。

1、if语句

这是一段最简单的if语句的代码：

int _tmain(int argc, _TCHAR* argv[])
{
    if(argc == 0)
    {
        printf("%d \r\n", argc);
    }
    return 0;
}

反汇编结果如下：

我们看到在cmp之后，有一个跳转指令JNE，判定结果为“jump not equal 即 不等于0则跳转”。竟然与我们的if语句的逻辑是相反的，为何不能将else的语句块提前放，到if语句块后面去呢？这样判定条件不就一致了吗？

非也！ 因为C语言是根据代码行的位置来决定编译后的二进制代码的地址高低的，也就是说，低数行对应低地址，高数行对应高地址。C语言的编译器不能随意改变代码行在内存中的顺序！

根据这个特征，我们将argc==0改为argc>0，则对应的汇编语言的指令应为“小于等于0”：

在反汇编中表达式短路和if语句这两种分支结构的实现是一样的，很难对他们进行区分。

总结：

执行影响标志位的指令
jxx         xxxx

一旦遇到这种格式的指令，需要高度怀疑它是由 [一个if语句组成的但分支结构]，根据比较信息与条件跳转指令，找到其跳转的相反的逻辑，即可恢复分支结构的原型。

2、if … else … 语句

我们改变一下上面这段代码：

int _tmain(int argc, _TCHAR* argv[])
{
    if(argc == 0)
        printf("argc == 0");
    else
        printf("argc != 0");
    return 0;
}

反汇编之后的代码如下：

与if类似，都是取相反的条件跳转指令，唯一不同的是，在else（地址 3413BB）处，多了一句jmp指令，直接跳到了函数的末尾。正常的流程跳转，不解释了。

总结

执行影响标志位的指令
jxx     ELSE_BEGIN             //这里是else的首地址
IF_BEGIN
...                            //这里是if代码块
IF_END
jmp     ELSE_END               //跳转到else语句的结束地址
ELSE_BEGIN
...                            //这里是else代码块
ELSE_END

遇到以上指令序列，先考察其中的两个跳转指令，当第一个跳转指令，跳转到地址ELSE_BEGIN处之前有一个JMP指令，则可认定为 if…else…组合了。

位操作符有以下几种：

  1. << 左移运算，最高位左移到CF中，最低位补零
  2. >> 右移运算，最高位不变，最低位右移到CF中
  3. | 或运算，相同位，有一个为1，则为1
  4. & 与运算，相同位，两个都为1，则为1
  5. ^ 异或运算，相同位，为异为真 即 如果不同则为1
  6. ~ 取反运算，将操作数的每一位取反，1变0,0变1

位运算一般在算法中大量使用，例如MD5算法，就是通过大量的位运算完成的。 如何使一个数不可逆呢？利用位运算就可以达到目的。 如： x & 0 = 0 根据结果，x的值是不可以逆推的。（取反 ~ 和 异或 ^ 可以反推）

下面这段代码用来演示位操作：

int main(int argc, int n)
{
    argc = argc << 3;
    argc = argc >> 5;
    argc = argc | 0xFFFF0000;
    argc = argc & 0x0000FFFF;
    argc = argc ^ 0xFFFF0000;
    argc = ~argc;
    return argc;
}

逆向基础第五篇：条件表达式

By John·Huang

发表于 2014-05-04

条件表达式又成为三目运算，语句结构如下：

<表达式1>?<表达式2>:<表达式3>;

条件表达式一般是先判断再选择，但是编译器不一定按这种方式编译，当表达式2和表达式3为常量时，条件表达式可以被优化；当表达式2或表达式3有一个为变量时，表达式不可以被优化，会变成分支结构。
当表达式1为常量时， 编译器编译期间将得到答案，将不会有表达式存在。

第一种方式

7:    int main(int argc, char* argv[])
8:    {
9:        return argc == 5 ? 5 : 6;
00401028   xor         eax,eax
0040102A   cmp         dword ptr [ebp+8],5
//setne检查ZF标记位，当ZF==1时，则赋值al为0，否则赋值al为1
0040102E   setne       al
//如果argc==5 则 al==0，否则 al == 1
00401031   add         eax,5
10:   }

上面的表达式2 和 表达式3的差值为1，使用setne指令平衡。

第二种方式

7:    int main(int argc, char* argv[])
8:    {
9:        return argc == 5 ? 4 : 10;
00401028   mov         eax,dword ptr [ebp+8]
0040102B   sub         eax,5
0040102E   neg         eax
00401030   sbb         eax,eax
00401032   and         eax,6
00401035   add         eax,4
10:   }
  • 执行sub指令后，eax 值就不为 0
  • 执行neg指令后，eax 的符号位就会发生改变，CF 置 1 。
  • 执行sbb eax,eax ，错位减法指令，等价于 eax = eax - eax - CF。当CF为1时，eax中的值将会是0xFFFFFFFF，否则为0 。
  • 使用 eax 与 6 做位运算后，如果eax中的数值原来为 -1，则记过为6，加4后得到数值 10
  • 当条件表达式 argc == 5 成立，那么eax中的值始终为 0 。用0与6做位运算结果还是0 ，加4后为4 。

总结：遇到 sub/neg/sbb 就表明是等值比较了，其判定值为A

sub        reg, A
neg        reg
sbb        reg, reg
and        reg, B
add        reg, C ;若等值条件成立，其结果为C，否则为B+C

这样的代码块可以直接还原成如下形式的高级代码：

reg == A ? C : B + C;

第三种方式

7:    int main(int argc, char* argv[])
8:    {
9:        return argc <= 8 ? 4 : 10;
//清空eax，与第一种方式类似
00401028   xor         eax,eax
0040102A   cmp         dword ptr [ebp+8], 8
//根据变量与8比较的结果，使用setg指令，当标记位SF=OF 且 ZF=0 赋值 al为1
//用于检查变量是否大于8，大于则赋值1
0040102E   setg        al
//此时al中只能是0或1，执行自减操作，eax中为 0xFFFFFFFF 或 0
00401031   dec         eax
//使用al与0xFA做位运算。 eax 中为 0xFFFFFFFA 或 0
//此数值为 表达式2 - 表达式3 的数值 
00401032   and         al,0FAh
//由于eax中只能有两个结果 0xFFFFFFFA(-6) 或 0，加 0x0A 后结果必然为 4 或 10 
00401034   add         eax,0Ah
10:   }

这种方式与第二种方式在逻辑上是相反的，它是 表达式2 减去 表达式3 得到的数值。

总结：先调整reg为0 或 -1

and         reg, A
add         reg, B

遇到这样的代码块，需要重点考察and前的指令，以判断真假逻辑的处理方式。 对于这个例子中的 dec reg 这样的指令，之前 reg 只能是 0 或 1，因此 这里的 dec 其实是对reg进行修正。 如果reg为1 修正后为0 ，否则为 0xffffffff，便于后面的and运算。 这时候要根据and前的指令流程，分析原来的判定在什么情况下会导致reg为0xffffffff 或 0 ，以便于还原。

第四种方式
当 表达式2 或 表达式3 的值未知时，编译器就会按常规流程处理了：

7:    int main(int argc, int n)
8:    {
9:        return argc ? 8 : n;
//比较变量argc
00401028   cmp         dword ptr [ebp+8],0
//如果argc值为0，则跳转到地址 00401037 处
0040102C   je          main+27h (00401037)
//如果argc不为0，则使用立即数8赋值，并跳转到地址0040103d处结束函数
0040102E   mov         dword ptr [ebp-4],8
00401035   jmp         main+2Dh (0040103d)
//取n的值并返回。
00401037   mov         eax,dword ptr [ebp+0Ch]
0040103A   mov         dword ptr [ebp-4],eax
//将结果放入eax，函数结束
0040103D   mov         eax,dword ptr [ebp-4]
10:   }

一、算术运算和赋值

赋值运算是将一个内存空间中的数据传递到另外一个内存空间中。由于内存没有处理器那样的控制能力，各个内存单元之间是无法传递数据的，必须通过处理器访问中转，需要借助寄存器实现赋值。

    nVar1 = 1 + 2;

    nVar1 = nVar1 + nVar2;

    return 0;

1.4、除法

在汇编中，idiv 和 div 分别为有符号数和无符号数的除法指令。C++中的除法不保留余数，有专门的求余数运算（% 取模）。 对于整数除法，C++中仅保留整数部分，小数部分完全舍弃。

在计算机中有 向下取整floor 、 向上取整ceil 和 向零取整。大多数语言的规定是向零取整，即 “截断除法”。

1.5、溢出

比如int类型的数据 0xFFFFFFFF加2得到的结果会超出 int类型的存储范围，超出的部分成为溢出。 溢出数据无法被保存，将会丢失。

进位
无符号数超出存储范围叫做 进位 。 因为没有符号，不会破坏数据，而多处的1位数据会被进位到标志位CF中。 可以通过查看进位标志位CF，检查数据是否进位。

溢出

有符号数超出存储范围叫做 溢出 。由于数据进位，会破坏有符号数的最高位 — 符号位。 可查看溢出标识 OF ，检查数据是否溢出。
