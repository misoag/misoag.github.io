逆向基础第十六篇：堆空间的申请与释放

By 鸡哥

发表于 2014-05-16

在C/C++中，使用malloc 与 new 实现堆空间的申请，返回的数据是堆空间的首地址。 对应地，使用free和delete完成堆空间的释放。但是需要堆空间的首地址。如果首地址丢失将无法释放堆空间，会导致内存泄露。

确定变量空间是否属于堆空间只要找到两个关键点即可：

  • 空间申请：malloc 与 new 等
  • 空间释放：free 与 delete 等

malloc,free 与 new,delete 对比：

int _tmain(int argc, _TCHAR* argv[])
{
    char * pCharMalloc = (char *) malloc(10);
    char * pCharNew =  new char[10];
    if(pCharNew != NULL)
    {
        delete [] pCharNew;
        pCharNew = NULL;
    }
    if(pCharMalloc != NULL)
    {
        free(pCharMalloc);
        pCharMalloc = NULL;
    }
}

逆向基础第十五篇：函数调用的三种约定方式（堆栈平衡）

By 鸡哥

发表于 2014-05-15

VC++ 环境下的的函数调用约定有三种：

  • _cedcl C/C++的默认调用方式，调用方平衡栈，不定长参数的函数可以使用。
  • _stdcall 被调用方平衡栈，不定长参数的函数无法使用
  • _fastcall 寄存器方式传参，被调用方平衡栈，不定长参数的函数无法使用。

1、_cdecl 与 _stdcall的对比

void _stdcall ShowStd(int nNumber)
{
    printf("%d",nNumber);
}

void _cdecl ShowCde(int nNumber)
{
    printf("%d",nNumber);
}

int _tmain(int argc, _TCHAR* argv[])
{
    ShowStd(5);//被调用方平衡栈
    ShowCde(5);//调用方平衡栈
}

main 函数反汇编代码如下：

    ShowStd(5);
00CC1A6E  push        5  
00CC1A70  call        ShowStd (0CC11C7h)  
    ShowCde(5);
00CC1A75  push        5  
00CC1A77  call        ShowCde (0CC101Eh)  
00CC1A7C  add         esp,4  //这里对栈进行了平衡

_cdecl 与 _stdcall 函数的反汇编代码对比：

经过对比，我们发现只有最后一行有区别。

_stdcall 反汇编最后ret 4 等价于：esp += 4 对栈顶加4，用于平衡栈顶。
_cdecl被调用后，在main函数中对esp进行了加4操作，用于平衡栈顶。

2、_fastcall 调用方式

测试代码：

void __fastcall ShowFast(int nOne,int nTwo,int nThree,int nFour)
{
    printf("%d,%d,%d,%d",nOne,nTwo,nThree,nFour);
}

int _tmain(int argc, _TCHAR* argv[])
{
    ShowFast(1,2,3,4);
}

反汇编代码：

    ShowFast(1,2,3,4);
00AB140E  push        4          ;使用栈方式传参数
00AB1410  push        3          ;使用栈方式传参数
00AB1412  mov         edx,2      ;使用寄存器传参
00AB1417  mov         ecx,1      ;使用寄存器传参
00AB141C  call        ShowFast (0AB11CCh)

由于寄存器的数量有限，而参数的数量有可能很多，因此fastcall仅使用了 edx和ecx，分别传递了第一个和第二个参数，其余的使用了栈传递参数。

逆向基础第十四篇：认识栈帧（堆栈平衡）

By 鸡哥

发表于 2014-05-14

当栈顶指针esp小于栈底指针ebp时，就形成了栈帧。在VC++中，栈帧用来存放局部变量、函数返回值地址、函数参数等。

当A函数调用B函数时，就会针对B函数开辟出所需的栈空间，形成此B函数的栈帧。当这个B函数结束调用时，要清除开辟出的栈空间，这一过程，称作 “堆栈平衡”。

下面是一个空函数：

int _tmain(int argc, _TCHAR* argv[])
{
    return 0;
}

反汇编代码如下：

int _tmain(int argc, _TCHAR* argv[])
{
00F01350  push        ebp                  ;保存寄存器ebp的值
00F01351  mov         ebp,esp              ;调整当前栈底指针到栈顶（将ebp作为栈底指针）
00F01353  sub         esp,0C0h             ;分配192字节的空间
00F01359  push        ebx                  ;保存寄存器ebx的值
00F0135A  push        esi                  ;保存寄存器esi的值
00F0135B  push        edi                  ;保存寄存器edi的值
00F0135C  lea         edi,[ebp-0C0h]       ;取栈顶地址，保存在edi中
00F01362  mov         ecx,30h  
00F01367  mov         eax,0CCCCCCCCh  
00F0136C  rep stos    dword ptr es:[edi]  
    return 0;
00F0136E  xor         eax,eax              ;对返回值eax清零
}
00F01370  pop         edi                  ;还原寄存器edi的值
00F01371  pop         esi                  ;还原寄存器esi的值
00F01372  pop         ebx                  ;还原寄存器ebx的值
00F01373  mov         esp,ebp              ;还原寄存器esp的值
00F01375  pop         ebp                  ;还原寄存器ebp的值
00F01376  ret
  • rep指令的目的是重复其上面的指令。
  • ecx的值是重复的次数。
  • stos指令的作用是，将eax中的值拷贝到es:[edi]指向的地址。

逆向基础第十三篇：编译器对循环的优化

By 鸡哥

发表于 2014-05-13

之前，我已经对do、while、和for循环做了详细的介绍。

do循环：http://www.0x520.com/2014/07/04/247.html
while循环：http://www.0x520.com/2014/07/04/251.html
for循环：http://www.0x520.com/2014/07/05/264.html

通过对比，我们已经知道，do循环的速度最快，while次之，for的速度是最慢的。 因为，do只用到一个跳转指令，while用到两个跳转指令，而for用到了三个跳转指令。

1、优化while循环

在 while循环 中我们讲过，是可以将while转换为do循环的，可以将do循环结构嵌套在if中，来完成while结构的优化。
while循环代码：

void loop_while(int loopTimes)
{
    while(loopTimes>=0)
    {
        //do something
        loopTimes --;
    }
}

优化代码：

void loop_do(int loopTimes)
{
    if(loopTimes>=0)
    {
        do{
            //do something
            loopTimes --;
        }while(loopTimes>=0);
    }
}

如果编译器使用O2选项进行优化，那么，反汇编之后的代码将是上面这种形式。

2、优化for循环

for循环，从结构上看，其结构特征和while循环类似。由于赋初值部分不属于循环体，可以忽略。只要将比较部分放入循环体，那么就是一个while循环。既然可以转换成while循环，那么必然也可以转换成do循环。

for循环代码：

int nSum = 0;
for (int i = 0;i < loop_times ; i++)
{
    nSum += i;
}

for转换成while循环：

int i = 0;
while(i<loop_times){
    nSum += i;
    i++;
}

while循环转换成do循环：

int nSum = 0;
int i = 0;
if(i<loop_times){
    do {
        nSum += i;
        i++;
    } while (i<argc);
}

编译器真的会这样优化吗？我们打开O2选项（选择Release，默认就是O2）：

编译如下代码：

int loop_for(int loop_times)
{
    int nSum = 0;
    for (int i = 0;i < loop_times ; i++)
    {
        nSum += i;
    }
    return nSum;
}

上图中for循环优化之后的结构一目了然，但是有些朋友表示还是看不懂，表示对下面这段代码不知所云：

add     edx,eax

这里我来讲解一下，其实这已经不是编译器对for循环优化的范畴了，这里是编译器对当前算法的优化 nSum += (i++);

  • edx 等于变量 nSum 的偶数和
  • ecx 等于变量 nSum 的奇数和
  • eax 等于变量 i

最后，在地址00401034处，lea eax,[ecx+edx]，这里 eax = 奇数和+偶数和 。

逆向基础第十二篇：控制流程 之 for循环

By 鸡哥

发表于 2014-05-12

for 循环是三种循环（do，while，for）中最复杂的一种。for循环由赋初值、设置循环条件、设置循环步长 这三条语句组成。在程序中for被使用的频率是最高的。

for循环测试代码：

{

    {

    }
    printf("%d \r\n",nSum);
    return 0;
}

反汇编代码如下：

通过上图，我们看出，赋初值只执行一次。然后不断自增、执行代码块，往上跳。通过循环流程，我们可以总结其特性。

总结：

Step1：赋初值，跳转到循环判定部分，然后执行代码块

Step2：代码块末尾，向上跳，跳转到自增（修改步长部分，这里是自增，视实际情况而定）
Step3：循环判定部分，然后执行代码块，向上跳跳到Step2

逆向基础第十篇：控制流程 之 do循环

By 鸡哥

发表于 2014-05-10

do循环工作流程清晰，识别相对简单。根据do循环的特性：先执行语句块，再进行比较判断。条件成立时，会继续执行语句块。C++中的goto也可以模拟do循环。

测试代码：

int _tmain(int argc, _TCHAR* argv[])
{
    int nSum    = 0;
    int nIndex    = 0;
    do 
    {
        nSum += nIndex;
        nIndex ++;
    } while (nIndex <= argc);
}

反汇编结果：

这与if分支结构非常相似，但是自己分析之后发现他们并不相同，这跟if语句的比较是相反的，if的跳转地址大于当前地址，是一个向下跳的过程。而do的跳转地址小于当前代码地址，是向上跳的。 根据这个特性，我们可以很容易的区分 if 和 do 的反汇编代码。

总结：

DO_BEGIN
......            ;循环语句块
;影响标记位的指令
jxx     DO_BEGIN

如果遇到上面这种代码块，可以判定为是一个do循环。根据跳转到的地址，可以得知代码块的首地址，jxx指令的地址为循环的结束地址。根据这些特征，即可还原do循环结构的原型。

逆向基础第九篇：流程控制 之 switch 判定树

By 鸡哥

发表于 2014-05-09

上一篇我们探究了switch的非线性索引表的优化，讲述了 case最大值 和 case最小值的差值在255以内的情况。

但是，当差值大于255时，这种优化方案会造成空间浪费。此时编译器采用另外一种优化方案——判定树。

判定树： 每个case作为一个节点，从这些节点中找出一个中间值作为根节点，以此形成一个平衡二叉树，以每个节点为判定值，大于和小于关系分别对应左子树和右子树，这样可以提高检索效率。

我们构造了这段代码：

switch(argc)
{
case 2: printf("The Number 1"); break;
case 3: printf("The Number 2"); break;
case 8: printf("The Number 3"); break;
case 10: printf("The Number 5"); break;
case 35: printf("The Number 6"); break;
case 37: printf("The Number 6"); break;
case 666: printf("The Number 255"); break;
}

1、switch 与 if…else if 反汇编比较

一个标准的switch结构：

【case地址表】

通过查表（数组访问），我们看到数组的第一个元素就是case1的首地址。以此类推，通过数组下标访问，我们可以更快速的找到其他case地址。

总结

jxx        DEFAULT_ADDR
jmp        dword ptr [eax*4 + xxxx]  ;地址xxxx为线性跳转表的首地址

根据跳转表中存放的地址，可以找到每个case的首地址，即可反推处线性关系的switch分支结构。

3、不能构成跳转表的switch（索引表）

在现实中，并非所有的switch都是有序线性的，当两个case的值间隔较大时，这样就会造成空间的巨大浪费。

对于非线性的switch结构，可以采用制作索引表的方法来进行优化。索引表优化需要两张表：case语句块地址表、case语句块索引表。

地址表中的每一项，保存case语句块的首地址，有几个case语句就有几项。default也在其中，如果没有，则保存一个switch的结束地址。

索引表中保存地址表的编号，它的大小等于 最大case值 减去 最小case值 的差值。 当差值大于255时，这种优化方案也会浪费空间。可以使用树方式优化。

索引表大小 = (MAX - MIN) * 1字节    //MAX=case的最大值，MIN=case的最小值
地址表大小 = SUM * 4字节            //SUM为case的数量
总占用字节 = 索引表大小 + 地址表大小

movezx 无符号扩展，并传送。一般用于将较小值拷贝到较大值中。

索引表（地址0BC35A0h处）截图

通过索引表我们可以看到，这里共有255个case值，会生成一个255字节大小的索引地址表，从6到255间隔了249个case值，这249项保存的是case地址表中switch结尾地址下标。

地址表（地址0BC3584h处）截图

索引表中存放的是地址表的下标，根据下标值可以找到对应case语句块的首地址。这种查询方式会产生两次间接内存访问，效率上要低于线性表方式。

总结：

mov        reg, mem                     //取出switch 变量
sub        reg, 1                       //数组下标平衡
mov        mem, reg
;影响标记位的指令
jxx        xxxx                         //超出范围的跳转到switch末尾，或default处
mov        reg, [mem]
xor        exa, eax
mov        al, byte ptr(xxx) [reg]      //查询索引表，得到地址表下标
jmp        dword ptr [eax * 4 + xxxx]   //查询地址表，得到对应case的首地址

遇到上面代码块，可判定为添加了索引表的switch结构。有两次查表过程，第一次是查询索引表，第二次查询地址表，可根据地址表的内容，恢复switch源码。

这篇文章中的代码是通过vs2010编译的，可能与你反汇编出来的结果不一致。

1、if语句

这是一段最简单的if语句的代码：

int _tmain(int argc, _TCHAR* argv[])
{
    if(argc == 0)
    {
        printf("%d \r\n", argc);
    }
    return 0;
}

反汇编结果如下：

我们看到在cmp之后，有一个跳转指令JNE，判定结果为“jump not equal 即 不等于0则跳转”。竟然与我们的if语句的逻辑是相反的，为何不能将else的语句块提前放，到if语句块后面去呢？这样判定条件不就一致了吗？

非也！ 因为C语言是根据代码行的位置来决定编译后的二进制代码的地址高低的，也就是说，低数行对应低地址，高数行对应高地址。C语言的编译器不能随意改变代码行在内存中的顺序！

根据这个特征，我们将argc==0改为argc>0，则对应的汇编语言的指令应为“小于等于0”：

在反汇编中表达式短路和if语句这两种分支结构的实现是一样的，很难对他们进行区分。

总结：

执行影响标志位的指令
jxx         xxxx

一旦遇到这种格式的指令，需要高度怀疑它是由 [一个if语句组成的但分支结构]，根据比较信息与条件跳转指令，找到其跳转的相反的逻辑，即可恢复分支结构的原型。

2、if … else … 语句

我们改变一下上面这段代码：

int _tmain(int argc, _TCHAR* argv[])
{
    if(argc == 0)
        printf("argc == 0");
    else
        printf("argc != 0");
    return 0;
}

反汇编之后的代码如下：

与if类似，都是取相反的条件跳转指令，唯一不同的是，在else（地址 3413BB）处，多了一句jmp指令，直接跳到了函数的末尾。正常的流程跳转，不解释了。

总结

执行影响标志位的指令
jxx     ELSE_BEGIN             //这里是else的首地址
IF_BEGIN
...                            //这里是if代码块
IF_END
jmp     ELSE_END               //跳转到else语句的结束地址
ELSE_BEGIN
...                            //这里是else代码块
ELSE_END

遇到以上指令序列，先考察其中的两个跳转指令，当第一个跳转指令，跳转到地址ELSE_BEGIN处之前有一个JMP指令，则可认定为 if…else…组合了。

3、if…else if…else if 多分支流程

继续改进上面的代码：

int _tmain(int argc, _TCHAR* argv[])
{
    if(argc > 0)
        printf("argc > 0");
    else if(argc == 0)
        printf("argc == 0");
    else
        printf("argc < 0");
    return 0;
}
