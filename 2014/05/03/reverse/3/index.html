    • 一、算术运算和赋值

    赋值运算是将一个内存空间中的数据传递到另外一个内存空间中。由于内存没有处理器那样的控制能力，各个内存单元之间是无法传递数据的，必须通过处理器访问中转，需要借助寄存器实现赋值。

    1.1、加法

    在VC++ 6.0 中编辑代码，如下：

    int main(int argc, char* argv[])
{
    int nVar1 = 0;
    int nVar2 = 0;

    nVar1 = nVar2 + 1;
    nVar1 = 1 + 2;

    nVar1 = nVar1 + nVar2;

    return 0;
}

    快捷键F10运行，然后选择Desassembly，（VS2010下右击菜单选择反汇编）如下图。

    编译器优化 之 常量传播

    编译期间可计算出的结果的变量转换成常量

    void main()
{
    int nVar = 1;
    printf("nVar = %d \r\n", nVar);
}

    变量nVar是一个在编译期间就可以计算出结果的变量。因此，程序中所有引用到nVar的地方都会被常量1替换。

    void main() 
{
    printf("nVar = %d \r\n", 1);
}
    编译器优化 之 常量折叠

    下面这段代码：

    int main(int argc, char* argv[])
{
    int nVar1 = argc;
    int nVar2 = argc;

    nVar1 = nVar1 + 1;
    nVar1 = 1 + 2;

    nVar1 = nVar1 + nVar2;
    printf("nVar1 = %d \r\n", nVar1);

    return 0;
}

    在Release模式（默认使用O2优化）下编译出EXE文件，用IDA分析之后，结果如下：

    ; int __cdecl main(int argc, const char **argv, const char **envp)
_main proc near

argc= dword ptr  8
argv= dword ptr  0Ch
envp= dword ptr  10h

push    ebp
mov     ebp, esp
mov     eax, [ebp+argc] // ebp + argc 是 ebp + 8 ，即“获取参数”
add     eax, 3
push    eax
push    offset Format   ; "nVar1 = %d \r\n"
call    ds:__imp__printf
add     esp, 8
xor     eax, eax
pop     ebp
retn
_main endp

    通过编译器计算，常量折叠之后的代码为：

    int main(int argc, char* argv[])
{
    printf("nVar1 = %d \r\n", 3 + argc);
    return 0;
}

    1.2、 减法

    在汇编中有sub指令用于减法计算，虽然计算机只会做加法，但是可以通过补码的转换将减法转变为加法完成。
    关于补码的知识可以参考我这篇文章：http://www.0x520.com/2014/07/01/175.html

    1.3、乘法

    乘法对应的汇编指令：有符号 imul 和 无符号mul 两种。由于乘法指令的执行周期较长，在编译过程中编译器会尝试将乘法变成加法，或使用移位等周期较短的指令。如果都不可转换时，才使用乘法指令。

    乘法测试代码如下：

    int main(int argc, char* argv[])
{
    int nVar1 = argc;
    int nVar2 = argc;
    //变量乘常量（常量值为非2的幂）
    printf("nVar1 * 15 = %d",nVar1 * 15);
    //变量乘常量（常量值为2的幂）
    printf("nVar1 * 16 = %d",nVar1 * 16);
    //常量相乘
    printf("2 * 2 = %d", 2*2);
    //混合运算
    printf("nVar2 * 4 + 5 = %d", nVar2 * 4 + 5);
    //两变量相乘
    printf("nVar1 * nVar2 = %d", nVar1 * nVar2);
    return 0;
}

    1.4、除法

    在汇编中，idiv 和 div 分别为有符号数和无符号数的除法指令。C++中的除法不保留余数，有专门的求余数运算（% 取模）。 对于整数除法，C++中仅保留整数部分，小数部分完全舍弃。

    在计算机中有 向下取整floor 、 向上取整ceil 和 向零取整。大多数语言的规定是向零取整，即 “截断除法”。

    1.5、溢出

    比如int类型的数据 0xFFFFFFFF加2得到的结果会超出 int类型的存储范围，超出的部分成为溢出。 溢出数据无法被保存，将会丢失。

    进位
    无符号数超出存储范围叫做 进位 。 因为没有符号，不会破坏数据，而多处的1位数据会被进位到标志位CF中。 可以通过查看进位标志位CF，检查数据是否进位。

    溢出

    有符号数超出存储范围叫做 溢出 。由于数据进位，会破坏有符号数的最高位 — 符号位。 可查看溢出标识 OF ，检查数据是否溢出。

    1.6、自增 和 自减

    测试代码如下：

    int main(int argc, char* argv[])
{
    int nVar1 = argc;
    int nVar2 = argc;
    nVar2 = 5 + (nVar1 ++);
    nVar2 = 5 + (++ nVar1);
    return 0;
}
